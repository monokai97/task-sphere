# Proposal: Hito 3.2 - Aplicación Visual Vento

## Porqué
Tras establecer los tokens de diseño, necesitamos integrar las extensiones estéticas de Vento (Glows, bordes sutiles, efectos de cristal/glassmorphism) en los componentes Shadcn base. Esto garantiza que la aplicación no solo sea funcional, sino que tenga la identidad visual premium definida en el diseño.

## Objetivo
Implementar los estilos de "Vento" en los componentes Shadcn instalados (Card, Button, Dialog) sin modificar el comportamiento de Radix, mediante clases extendidas de Tailwind y capas CSS.

## Impacto
- Identidad visual única de "Vento".
- Experiencia de usuario coherente y premium.
- Elevación de los componentes base hacia una estética de "Enterprise Guest-First".
